### Decisões tomadas
1. InstallPrompt é sanctuary incondicional (sem seletor redesign) — usuários novos chegam via landing, sempre veem sanctuary antes da flag
2. LLM proibido de usar markdown no prompt (Option A) em vez de implementar markdown renderer — mais simples para persona Dona Maria
3. `var(--color-primary, #006a5e)` fallback NÃO funciona quando `--color-primary` está definido como rosa no `:root` legado — usar hexadecimal hardcoded
4. SANCTUARY_COLORS extraído para `confettiColors.js` separado — ESLint `react-refresh/only-export-components` proíbe export de constante + default component no mesmo arquivo
5. DLQ error ao carregar é limitação de infra pré-existente (Vite dev não serve Vercel serverless) — não é bug W14

### Lições aprendidas
- ESLint react-refresh bloqueia exportar constante + componente default no mesmo arquivo
- `var(--token, fallback)` nunca usa o fallback se o token está definido (mesmo com valor errado no `:root` legado)
- btn-secondary override no redesign precisava de `box-shadow: none` para remover glow neon da versão legada
