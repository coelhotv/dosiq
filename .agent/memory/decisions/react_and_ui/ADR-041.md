## Decisão

No workspace `apps/web`, a regra de lint `react-native/no-inline-styles` (ou equivalente para React Web) permanecerá desativada ou configurada como 'warning', permitindo a coexistência de estilos inline legados com o novo sistema de tokens do "Santuário Terapêutico".

## Rationale

- **Problema:** O redesign incremental do Dosiq Web gerou uma arquitetura híbrida onde componentes legados dependem fortemente de inline styles para ajustes finos de layout.
- **Contexto:** Durante a Fase 4 de Zero-Error Lint Hygiene, identificou-se que a conversão total de inline styles para CSS Classes/Tokens exigiria refatoração estrutural de alto risco fora do escopo de estabilização.
- **Impacto:** Manter os estilos inline permite foco na estabilidade funcional e lógica, adiando a purificação estética para sprints específicas de UI.

## Solução

1. **Prioridade de Erro:** Focar na eliminação de erros bloqueantes (Console, Unused Vars, Render Loops) em vez de avisos de estilo.
2. **Migração sob Demanda:** Novos componentes DEVEM seguir o sistema de tokens. Componentes existentes são migrados apenas se houver necessidade de alteração lógica.
3. **Padrão de Bypass:** Uso de `style={{ ... }}` é aceito no Web sem necessidade de comentários de supressão por linha, para evitar ruído visual no código.

## Implementação

- **Lint:** Configuração centralizada no `.eslintrc.js` para tratar `no-inline-styles` como aviso.
- **Documentação:** Este ADR serve como justificativa para revisores e agentes futuros.
