## Specification

Todos os canais de notificação DEVEM retornar um objeto com a seguinte forma:

```typescript
ChannelResult = {
  channel: 'telegram' | 'mobile_push',        // Identifica qual canal executou
  success: boolean,                            // true se pelo menos uma tentativa foi bem-sucedida; false se todas falharam
  attempted: number,                           // Total de tentativas feitas (0 se nenhum device/chat disponível)
  delivered: number,                           // Total de entrega com sucesso
  failed: number,                              // Total de falhas
  deactivatedTokens: string[],                 // Tokens permanentemente desativados devido a erro (ex: DeviceNotRegistered)
  errors: Array<{                              // Array de erros (se vazio, success=true)
    message?: string,
    code?: string,                             // Ex: 'DeviceNotRegistered', 'InvalidCredentials'
    token?: string,                            // Para erros por device (Expo)
  }>,
}
```

## Invariants per Channel

### Telegram Channel

| Condition | attempted | delivered | failed | success | deactivatedTokens |
|-----------|-----------|-----------|--------|---------|-------------------|
| telegram_chat_id existe, envio OK | 1 | 1 | 0 | true | [] |
| telegram_chat_id não existe | 0 | 0 | 0 | true | [] |
| telegram_chat_id existe, envio falha | 1 | 0 | 1 | false | [] |

**Notas:**
- `attempted = 0` quando `telegram_chat_id = null` (NÃO é erro, é configuração pendente)
- `deactivatedTokens` SEMPRE vazio para Telegram (não há conceito de "token permanente" no Telegram)
- Erros de rede/API vão para `errors[]` mas não desativam nada

### Expo Push Channel

| Condition | attempted | delivered | failed | success | deactivatedTokens | errors |
|-----------|-----------|-----------|--------|---------|-------------------|--------|
| 0 devices ativos | 0 | 0 | 0 | true | [] | [] |
| 2 devices, ambos OK | 2 | 2 | 0 | true | [] | [] |
| 2 devices, 1 OK + 1 DeviceNotRegistered | 2 | 1 | 1 | false | ['ExponentPushToken[bad]'] | [{code: 'DeviceNotRegistered', token: '...'}] |
| expoClient.send() rejeita (timeout) | N | 0 | N | false | [] | [{message: 'timeout'}] |

**Notas:**
- `attempted = N devices` onde N >= 0
- `deactivatedTokens` contém APENAS erros permanentes (DeviceNotRegistered, InvalidCredentials, MessageTooBig)
- Erros transitórios (timeout, 5xx) vão para `errors[]` SEM desativar token
- Se expoClient rejeita completamente, `attempted = devices.length`, `delivered = 0`, `failed = devices.length`

## Normalization

`normalizeChannelResults(results: ChannelResult[])` consolida N resultados:

```typescript
{
  success: boolean,              // true se todos os canais tiveram success=true
  channels: ChannelResult[],
  totalDelivered: sum(channels.map(c => c.delivered)),
  totalFailed: sum(channels.map(c => c.failed)),
}
```

## Breaking Changes

- Remover `deactivatedTokens` ou alterar seu tipo
- Remover `errors` array
- Alterar `channel` para um valor não-previsto
- Mudar semântica de `success` (ex: true se nenhuma falha transitória)

## Non-Breaking Changes

- Adicionar novo campo opcional a ChannelResult
- Adicionar novo código de erro a `errors[].code`
- Renomear `telegram_chat_id` não afeta este contrato (é detalhe interno de telegramChannel)
