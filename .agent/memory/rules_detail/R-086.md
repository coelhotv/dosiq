# R-086 — Serverless Response Format [HIGH]

## Rule
In Vercel serverless functions, always use `res.status(code).json(body)` format.

## Pattern
```javascript
// CORRECT
return res.status(200).json({ success: true, data })

// WRONG — Express style, may not work correctly in Vercel
return res.json({ success: true, data })
```

## Source
Sprint 7 (PR #152) — response headers not sent correctly

## Consolidated anti-patterns
- `AP-S05` — return serverless responses in inconsistent format

Use this rule as the canonical guidance for Vercel response structure. The AP remains as a compact anti-example.
