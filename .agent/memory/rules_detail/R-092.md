# R-092: Spec Path Verification Before Edit

**Summary:** Before editing any file referenced in a spec, always verify the actual path with find. Specs often reference canonical paths that differ from reality.

**Applies to:** all

**Tags:** file-management, safety

## Rationale

Specs can drift from the codebase. Verifying the real file path before editing prevents changes landing in the wrong file or in a stale path.

## Implementation Notes

Confirm the actual target path with `find`, `rg --files`, or equivalent before starting edits.
