# R-106: Testes com Date Devem Usar setHours (não UTC Hardcoded)

**Summary:** Ao criar datas de referência em testes com horas locais, NUNCA usar timestamps UTC fixos — produzem horas locais diferentes por timezone. Use setHours() local.

**Applies to:** all

**Tags:** testing, timezone

## Rationale

Hardcoded UTC timestamps behave differently across environments. Local-hour tests should define local time directly.

## Implementation Notes

Create the date locally and then apply `setHours()` rather than embedding fixed UTC strings.
