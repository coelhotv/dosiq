# R-094: CSS Selector Over getByText for Non-Unique Text

**Summary:** When asserting on text that may appear in multiple DOM elements, use container.querySelector('.class').textContent instead of screen.getByText().

**Applies to:** react

**Tags:** testing, react

## Rationale

Repeated text values make `getByText` ambiguous and flaky. Stable selectors make tests more deterministic when the same copy appears in multiple places.

## Implementation Notes

Prefer a stable class or container-scoped selector whenever a text assertion could match more than one element.
