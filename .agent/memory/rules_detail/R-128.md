# R-128: Promise Chaining vs Async/Await

**Summary:** Prefer async/await for readable sequential logic

**Applies to:** ts

**Tags:** process, testing

## Rationale

async/await is more readable and handles errors naturally. Promise chains become unreadable with deep nesting.
