# R-031 — MarkdownV2 Escaping [HIGH]

## Rule
Always use `escapeMarkdownV2()` for messages with `parse_mode: 'MarkdownV2'`.

**Escape order:** backslash FIRST, then all 17 other special characters: `_ * [ ] ( ) ~ ` # + - = | { } . !`

Do NOT escape text in `answerCallbackQuery` — it uses plain text.

## Why
DLQ errors from unescaped `!` in MarkdownV2 messages. Telegram rejects the entire message if any special character is unescaped.

## Characters requiring escape (in MarkdownV2)
`\ _ * [ ] ( ) ~ ` # + - = | { } . !`

## Consolidated anti-patterns
- `AP-009` — send MarkdownV2 text without escaping

Use this rule as the canonical guidance for Telegram Markdown safety. The AP remains as a quick negative reference.
