# R-110: Data Freshness Guarantees

**Summary:** Document and enforce data freshness requirements for each endpoint

**Applies to:** ts, supabase

**Tags:** supabase, performance

## Rationale

Some data can be cached for hours; some must be real-time. Explicit freshness requirements prevent stale-data bugs and unnecessary API calls.
