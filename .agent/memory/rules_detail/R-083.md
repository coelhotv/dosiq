# R-083 — Environment Variable Fallbacks [HIGH]

## Rule
When accessing env vars that may have alternative names across environments (local vs CI vs production), always provide fallbacks.

## Common Scenarios
- `VITE_*` prefix for Vite-exposed variables vs unprefixed for server-side
- Different naming between CI and production
- Legacy variable names still in some environments

## Pattern
```javascript
// CORRECT
const SUPABASE_URL = process.env.SUPABASE_URL || process.env.VITE_SUPABASE_URL

// WRONG — fails if only VITE_SUPABASE_URL exists
const SUPABASE_URL = process.env.SUPABASE_URL
```
