# R-116: Console Logs for Development Debugging

**Summary:** Manter logs detalhados é obrigatório. Cada operação crítica deve ter console.log/error com contexto detalhado. Não remover logs ao mergear.

**Applies to:** all

**Tags:** process

## Rationale

This project uses logs as an active debugging aid during development and operations. Removing high-signal logs too early reduces observability.

## Implementation Notes

Keep meaningful logs with actionable context. Avoid changing or removing them casually during unrelated work.
