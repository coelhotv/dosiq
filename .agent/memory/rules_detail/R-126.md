# R-126: Serializar Queries Supabase em Views Mobile

**Summary:** Views mobile-critical NUNCA devem disparar >4 requests Supabase simultâneos. Usar fases: UI-blocking (max 2) → requestIdleCallback → IntersectionObserver.

**Applies to:** supabase, react

**Tags:** performance, supabase

## Rationale

Excessive parallel requests hurt mobile responsiveness and first-load stability.

## Implementation Notes

Phase data loading so only the most important requests block the first paint.
