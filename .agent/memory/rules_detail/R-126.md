# R-126: Error Recovery Strategies

**Summary:** Define recovery behavior for transient vs permanent errors

**Applies to:** ts

**Tags:** process, testing

## Rationale

Not all errors warrant retries. Distinguish transient (retry) vs permanent (fail) errors and implement appropriate recovery.
